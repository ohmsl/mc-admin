## Design Context

### Users
The primary users are trusted community staff and moderators in a homelab-hosted Minecraft setup.
They access the panel during active server administration to monitor current player activity and perform low-friction operational actions safely.

### Brand Personality
Industrial, practical, and disciplined.
The interface should feel like an operations console: fast to scan, explicit about risk, and clear about who did what.

### Aesthetic Direction
Industrial command-center layout using default shadcn component styling and semantic tokens.
No direct edits to shadcn component source files; composition, hierarchy, spacing, and information architecture drive the visual tone.
Desktop-first with mobile-safe adaptation.

### Design Principles
1. Prioritize operational clarity over decorative flourish.
2. Keep dangerous actions explicit and intentionally gated.
3. Surface current server state quickly (reachability, players, recency of checks).
4. Preserve small backend/API surface and map UI actions to constrained commands.
5. Maintain traceability through visible audit context after every action.
